# ccRecall 小孩版介紹

> 用十歲小孩能懂的方式，講 ccRecall 是什麼、怎麼運作。
> 建立日期：2026-04-17

---

## 故事背景

你有一個超聰明的朋友叫 Claude，會寫程式、解難題，但他是金魚腦——每次見完面，下次見到你就忘光光。昨天一起破解的謎題，明天又變陌生人。

## ccRecall 是什麼？

就是幫 Claude 裝上「記憶」的小幫手。分五步驟工作：

### 第一步：偷看日記

Claude 每次跟你聊天，電腦會自動把對話寫成日記檔（叫 JSONL，像一行一行的小紙條）。我派兩個助手：Scanner 負責找日記放在哪，Parser 負責把亂七八糟的格式翻譯成看得懂的字。

### 第二步：畫重點

日記超長，整本讀完要一小時。我有個叫 Summarizer 的工具，像老師用螢光筆畫重點——看到「我們決定用這個方法」「終於修好這個 bug」這種關鍵句就留下，廢話丟掉。而且它不靠 AI，是用一條條規則判斷，又快又省錢。

### 第三步：做成索引卡

畫完的重點，我存進一個叫 SQLite 的小倉庫，再給每個字加上「超快搜尋」功能（叫 FTS5）。這樣 Claude 想找「上次我們怎麼解那個 bug」，一秒就翻到。

### 第四步：遞小紙條

Claude 開始新對話時，我悄悄遞一張小紙條給他：「記得喔，上次你們決定用 A 方案、修好了 B 問題。」紙條超短（不到 300 個字），不會塞爆他腦袋。這個遞紙條的通道叫 MCP，就像 AI 之間的對講機。

### 第五步：自動發動

最棒的是 Claude 不用自己動手。有個叫 Hooks 的小鬧鐘，他一開始聊天，鬧鐘就響，自動叫我遞紙條。

## 一句話總結

就像電動遊戲的「存檔功能」——我幫 Claude 把過去的冒險記錄下來，下次見面他能從上次離開的地方繼續玩。

## 技術名詞對照表（給大人看）

| 小孩版 | 技術名詞 | 說明 |
|--------|---------|------|
| 日記檔 | JSONL | Claude Code 對話記錄格式，一行一個 JSON |
| 找日記的助手 | Scanner | 掃描 `~/.claude/projects/` 找出所有 session |
| 翻譯員 | Parser | 解析 JSONL，寬容模式（未知結構保留 raw） |
| 畫重點的老師 | Summarizer | 規則式摘要引擎（不靠 LLM） |
| 小倉庫 | SQLite | 本地資料庫，better-sqlite3 |
| 超快搜尋 | FTS5 | SQLite 內建的全文搜尋引擎 |
| 對講機 | MCP | Model Context Protocol，AI 服務間通訊協定 |
| 小鬧鐘 | Hooks | Claude Code 的事件觸發機制 |
| 小紙條 | Context Injection | 注入最相關片段（<300 tokens） |
